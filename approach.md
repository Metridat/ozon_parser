# approach.md — лог подхода

## Что пробовали и почему не сработало

**Попытка 1: requests + BeautifulSoup**
Не подходит — Ozon это React SPA, HTML приходит пустым.
Данные грузятся через JS после загрузки страницы.

### Попытка 2: Playwright headless=False без антидетекта
**Результат:** 403 Forbidden мгновенно.
**Причина:** DataDome видит Chromium по TLS fingerprint
ещё до проверки User-Agent.

### Попытка 3: Playwright + антидетект аргументы без прокси
**Результат:** всё равно 403.
**Причина:** DataDome анализирует десятки сигналов —
Canvas fingerprint, WebGL, поведение мыши.
Обычный Playwright она видит насквозь.
Забегу вперед - здесь неверно, с playwright_stealth и
грамотным user_agent - ozon пропускает

### Попытка 4: Playwright + playwright-stealth v2 + SOCKS5 прокси (Happ)
**Результат:** 200, страница загружается, заголовок "термос - купить на OZON".
**Почему сработало:** residential IP через Happ меняет репутацию
соединения, stealth патчит JS объекты (navigator.webdriver и др).

### Попытка 5: headless=True с прокси
**Результат:** total_checked=0, товары не перехватываются.
**Причина:** DataDome блокирует headless Chromium по дополнительным
сигналам — Canvas fingerprint, WebGL, отсутствие GPU рендеринга.
**Решение:** оставил headless=False.
В продакшене решается через Camoufox или Rebrowser
(антидетект браузеры с патченным fingerprint).

## Как нашли структуру данных

1. Перехватывали все XHR через page.on("response")
2. Нашли нужный endpoint: entrypoint-api.bx/page/json/v2
3. Данные о товарах лежат в widgetStates["tileGridDesktop-*"]
   как JSON-строка внутри JSON (двойная сериализация)
4. Поле с артикулом: item["sku"] (число, конвертируем в строку)
5. Изначально фильтровали по "skuId" — не находили.
   Ozon хранит артикул в двух местах:
   - item["sku"] — позиция в выдаче (нужное нам)
   - multiButton.ozonButton.skuId — кнопка корзины (не то)

## Проблема персонализации выдачи

Ozon меняет порядок выдачи между запросами даже с одним запросом.
SKU который был на позиции 1 — через 30 секунд может не попасть
в топ-24. Это поведение платформы, не баг парсера.

Как обнаружили: брали SKU из resp_0.json, через минуту
он уже не находился. Оказалось — выдача персонализируется
по IP, сессии, времени суток.

В продакшене решается:
- Авторизацией под фиксированным аккаунтом Ozon
- Параметром deny_category_prediction=true в URL
- Фиксированным регионом в куках

## Проблема total_checked < 100

Изначально скроллили фиксированно до y=2400 — получали 24 товара.
Задание требует проверить 100 позиций.

Решение: цикл скролла до document.body.scrollHeight
с проверкой len(all_skus) < 100.
Ozon подгружает по 12 товаров за скролл.
При 8-10 скроллах получаем 96-108 товаров.

## Итоговые лимиты которые считаю безопасными

- 1 запрос / 30 сек с одного IP — безопасно
- 1 запрос / 10 сек — допустимо для коротких сессий
- Без паузы — блокировка через 3-5 запросов
- Максимум 200 запросов в день с одного IP


## Финальный вывод по антидетекту

Ключевой компонент — playwright-stealth, а не прокси.
Stealth патчит navigator.webdriver и другие JS объекты которые
DataDome проверяет в первую очередь.

Прокси нужен только если домашний IP уже помечен.
В коде прокси закомментирован — скрипт работает без него.
Если получаете 403 — раскомментируйте строку с proxy.
