# architecture.md — масштабируемая архитектура

## Задача
500 товаров × 20 запросов = 10 000 проверок в день.
Алерт в Telegram если позиция упала ниже 50.
Хранение в PostgreSQL.

## Технологии и почему

**Celery + Redis** — очередь задач.
10 000 проверок / 24 часа = ~7 проверок в минуту.
Каждая проверка ~30 сек → нужно 4-5 воркеров параллельно.
Celery позволяет масштабировать воркеры горизонтально.

**Playwright на воркерах** — тот же стек что уже работает.
Каждый воркер держит один браузер, делает запрос, закрывает.

**PostgreSQL** — хранение результатов.
Схема простая:
- таблица `checks` (id, sku, query, position, checked_at)
- таблица `products` (sku, name, query)
- индекс по (sku, checked_at) для быстрых запросов истории

**Прокси-пул** — 20-50 residential IP с ротацией.
Провайдер: Brightdata или Smartproxy, ~$50-100/мес.
Ротация каждые 10 запросов или при получении 403.

**Telegram Bot** — алерт при падении позиции ниже 50.
python-telegram-bot, проверка после каждой записи в БД.

## Архитектура
```
Scheduler (Celery Beat)
    ↓ каждый час добавляет задачи в очередь
Redis Queue
    ↓
Воркеры (4-5 штук, Celery Worker)
    ↓ каждый воркер
Playwright + Proxy Pool
    ↓ результат
PostgreSQL
    ↓ если position > 50
Telegram Alert
```

## Устойчивость

**Ротация прокси:** пул 50 IP, смена каждые 10 запросов.
При 403 — автоматическая смена IP и повтор через 60 сек.

**Rate limiting:** не более 1 запроса в 30 сек с одного IP.
При 10 воркерах и 50 IP = безопасно до 100 запросов в минуту.

**Retry:** 3 попытки с экспоненциальной паузой (30, 60, 120 сек).
Если все 3 упали — задача помечается failed, алерт в Telegram.

**Мониторинг:** Flower (UI для Celery) показывает состояние воркеров.

## Риски и как снижаем

| Риск | Вероятность | Решение |
|------|-------------|---------|
| Ozon меняет структуру API | Средняя | Тесты на парсинг, алерт при падении success rate |
| Блокировка IP пула | Средняя | Residential прокси, ротация, человекоподобные паузы |
| Персонализация выдачи | Высокая | Фиксированный аккаунт Ozon или анонимные сессии |
| Падение воркера | Низкая | Celery автоматически перезапускает задачу |
